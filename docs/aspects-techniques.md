# Aspects techniques / Architecture

## Architecture
- Frontend React + TypeScript + Vite + Tailwind pour une expérience rapide et accessible.
- Backend Supabase avec PostgreSQL, politiques RLS et Edge Functions pour les traitements serverless.
- Moteur IA basé sur Gemini pour la génération, le résumé et la catégorisation des tickets.

## Sécurité et conformité
- Séparation multi-entreprises par RLS, gestion des rôles (utilisateur, agent, manager) et permissions fines.
- CSP stricte, bonnes pratiques RGPD et hébergement Supabase pour la gouvernance des données.
- Journaux et contrôles d’accès prévus pour tracer les actions critiques.

## Accessibilité
- Respect des principes RGAA 4.1 : navigation clavier, focus visible et ARIA cohérente.
- Contrastes vérifiés, prise en charge des assistances vocales/lecteurs d’écran et alternatives textuelles.
